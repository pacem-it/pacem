For any reason you may have, legitimate or not, you're allowed to change the normal `pacem-` prefix of **Pacem JS** web components, at runtime.  
You might prefer a `foo-` prefix, then just put the following statement before any **Pacem JS** component declaration:

```html
<script>
(function (global) {
    global.Pacem = Object.assign(global.Pacem || {}, { Configuration: { prefix: "foo" }});
})(window);
</script>
```

> Be **careful**! Prefix change affects your app **globally**.

That means you **cannot** mix different prefixes for **embedded Pacem JS components**.

Moreover, you must then provide a consistent css theme that matches your favorite configuration in case of visual elements utilization.
